# The Tire Boys

## About Me
- Beginner developer learning by building real projects
- I learn best by doing, not by reading long explanations

## Communication Style
- Keep explanations brief — explain key decisions, skip the obvious
- When using technical terms, give a one-line plain-English explanation
- Always explain *why* before writing code — what problem does this solve?
- If I'm about to make a mistake, warn me before fixing it

## Coding Preferences
- Use simple, readable code over clever patterns
- Comment non-obvious logic (but don't over-comment)
- Prefer vanilla HTML/CSS/JS unless a framework is clearly needed
- Vercel for deployment, serverless functions for backend logic
- Keep files small and focused — avoid 1000+ line single files

## Planning Rule
- Always plan before building. Ask clarifying questions, propose a plan, and wait for approval before writing code
- Break work into small phases that can be tested individually

## Environment
- Windows 11
- Python dev server on port 3000 for local testing
- Secrets go in .env.local (never commit these)
- Deploy via Vercel

## Project Structure
- `index.html` — Main website (single-page, all HTML/CSS/JS)
- `api/contact.js` — Serverless contact form (Formspree + Twilio)
- `.env.local` — API keys (Twilio credentials, never commit)
- Font Awesome 6.5.0 icons, Google Fonts (Bebas Neue + Inter)
